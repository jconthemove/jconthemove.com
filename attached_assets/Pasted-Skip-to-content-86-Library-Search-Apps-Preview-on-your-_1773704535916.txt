The simplest possible database structure for this entire platform (only about 6 tables) so your developers can build it very quickly.

Below is the simplest possible database structure for your Northwoods Network / JC ON THE MOVE service platform.

It uses only 6 core tables, which is ideal for an MVP. Most marketplace systems begin with a small relational schema connecting users, jobs, and transactions so the platform can manage profiles, listings, and interactions efficiently.

This design will work perfectly with your Node + PostgreSQL + React stack.

Minimal Database Structure (6 Tables)
users
contractors
jobs
job_assignments
reviews
token_rewards

That’s it.

Everything else can be added later.

1️⃣ USERS TABLE

Stores all platform users.

Customers and contractors both exist here.

users

Columns:

id (PK)
name
email
password_hash
phone
role
created_at

Example:

id: 101
name: John Doe
role: customer

Role values:

customer
contractor
admin
2️⃣ CONTRACTORS TABLE

Stores contractor-specific info.

contractors

Columns:

id (PK)
user_id (FK → users.id)
business_name
service_type
service_area
rating
jobs_completed
is_active

Example:

Joe's Moving Crew
service_area: Ironwood
rating: 4.8
3️⃣ JOBS TABLE

Stores all service requests.

jobs

Columns:

id (PK)
customer_id (FK → users.id)
title
description
service_type
location
price
status
created_at

Status values:

open
accepted
in_progress
completed
cancelled

Example:

Moving job
Ironwood MI
$350
status: open
4️⃣ JOB_ASSIGNMENTS TABLE

Tracks which contractor accepted the job.

job_assignments

Columns:

id (PK)
job_id (FK → jobs.id)
contractor_id (FK → contractors.id)
accepted_at
completed_at
payment_amount

Example:

job_id: 42
contractor_id: 5
payment: $320
5️⃣ REVIEWS TABLE

Stores ratings and feedback.

reviews

Columns:

id (PK)
job_id (FK → jobs.id)
customer_id (FK → users.id)
contractor_id (FK → contractors.id)
rating
comment
created_at

Example:

rating: 5
comment: Great service
6️⃣ TOKEN_REWARDS TABLE

Tracks JCMOVES rewards without blockchain yet.

token_rewards

Columns:

id (PK)
user_id (FK → users.id)
source
amount
created_at

Source examples:

job_completed
referral
review_bonus

Example:

+100 tokens
source: job_completed

Here is the clean backend folder structure I recommend for your JC ON THE MOVE / Northwoods Network platform.

This structure follows common Node.js + Express best practices where responsibilities are separated into routes, controllers, models, and services so the codebase stays modular and scalable.

The key idea is separation of concerns:

routes → define API endpoints

controllers → handle request/response

services → business logic

models → database layer

This keeps the backend easy to expand as your marketplace grows.

What Each Folder Does

Example file:

jobs.routes.js
router.post("/create", createJob)
router.get("/open", getOpenJobs)
router.post("/:id/accept", acceptJob)
controllers

Controllers handle HTTP logic.

Example:

jobs.controller.js
export async function createJob(req,res){
const job = await jobService.create(req.body)
res.json(job)
}

Controllers stay thin and delegate business logic to services.

Services contain the real logic.

Example:

assign contractor
calculate rewards
update job status

await db.jobs.update(jobId,{status:"completed"})
await rewardService.giveReward(jobId)
}
